Git

Git og GitHub er versjons og kontrollsystem, og disse systemene er godt egnet for sammarbeid og det er en mer strukturert måte å dele koder med hverandre, og det lagrer filene strukturert, og det er egnet seg til gjenoppretting av filer du kanskje har slettet på din egen private maskin.

Git er det verktøyet vi bruker til versjonskontroll og kodedeling, vi kan lagre alt vi vil og hvordan vi har utviklet kodene våre her, og vi kan også gå tilbake til hvilket som helst commit-punkt. Grunnen til at Git er så bra, er at om vi sitter på ulike steder og skal jobbe med samme prosjekt vil vi i prinsipp ikke møte på noe problemer. Git har mulighet til å løse mange problemer vi kan støte på av forskjellige koder på ulike maskiner, om du støter på problemer (som kan skje en sjelden gang, kan vi gjøre dette manuelt, og det finnes ulike hjelpemidler slik at vi kan få best mulig hjelp og det er blandt annet Eclipse eller IntelliJ IDEA. Men det som er fint med Git er at du kan gjøre små feil uten at det kan ha store konsekvenser, men er lurt å være litt forsiktig og veldig lurt å prøve frem kodene før du comitter arbeidet slik at ikke alt arbeidet blir ødelagt for alle.

GitHub

Github er et koding-hosting plattform for versjonskontroll og kan brukes til sammarbeid slik at det går ann å jobbe på samme prosjekt overalt. Github er en populær måte å kunne opprette og vurdere kode. For å bruke GitHub er vi nødt til å lage en bruker, og deretter kan vi starte morroa med det hele, og vi kan nå gjøre det vi måtte ønske av koding.
